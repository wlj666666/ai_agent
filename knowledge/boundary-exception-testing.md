# 边界值与异常值测试规范

## 边界测试原则

边界测试必须覆盖阈值本身、阈值两侧（略低于与略高于）以及边界外的代表值。

## TTC 边界（1.5 秒）

针对 TTC 小于等于 1.5 秒（TTC <= 1.5 秒）的边界规则，必须测试阈值本身和阈值两侧，至少包含以下用例：

| 场景 | TTC 值 | 预期 |
|------|--------|------|
| 阈值上侧（不触发） | 1.51 | 不触发制动 |
| 阈值点（触发） | 1.50 | 触发制动 |
| 阈值下侧（触发） | 1.49 | 触发制动 |
| 零边界 | 0.0 | 按规范处理 |
| 极小正值 | 0.01 | 触发制动 |

**注意**：等号方向（<=）容易遗漏，必须显式测试 1.50 与 1.51 两个相邻值。

## 相对速度边界

- 略大于 0（如 0.01 m/s）：应触发（配合合法 TTC）。
- 等于 0：不应触发。
- 略小于 0（如 -0.01 m/s）：不应触发。

## 异常输入规范

以下输入必须单独设计异常用例，预期为不触发或抛出/返回错误：

- TTC 为负数、NaN、None 或非数值类型。
- 相对速度为 NaN 或非数值类型。
- sensor_valid 缺失或非布尔类型。
- 所有字段同时无效的组合输入。

## 传感器无效场景

当 sensor_valid 为 False 时，即使 TTC 与相对速度满足触发数值，也不得触发制动。
